# Security and privacy

The shipped spec accepts only public HTTPS sources. Specs are strict (unknown and incomplete fields are rejected), and source responses are timeout-, byte-, item-, and source-bounded. The byte limit is enforced while reading the response stream, before UTF-8 decoding or parser buffering, including responses without `Content-Length`. Evidence excludes headers and stores only a short plain-text title/summary, provenance URLs, retrieval time and digest. Source URLs must use public HTTPS literals, credentials are rejected, and redirects are not followed.

HTTP and MCP request bodies are capped while streaming, including chunked bodies; an oversized declared `Content-Length` is rejected early. `render_markdown` accepts only a complete validated `Brief`. The reusable `createHttpApp()` has no default runner: execution is opt-in and must be injected behind a deployment policy. The public Worker's runner accepts only the exact checked-in example. Responses include a baseline CSP, frame denial, MIME-sniffing prevention, referrer policy, and permissions policy.

Proposals always carry `approvalRequired: true`; Orbit never acts on them. The MCP surface has no action, credential, filesystem, persistence, or arbitrary URL-fetch tool. MCP tool errors are returned as data and do not disclose headers. The local CLI writes only to its requested directory. Before accepting untrusted hosted specs, require authentication such as Cloudflare Access, add DNS-resolution/egress protection against rebinding, isolate credentials per adapter, add retention/deletion policy, rate limits and audit approval transitions. Treat feed text as untrusted data; deterministic lenses do not execute or follow its instructions. Review legal/robots and source terms.
